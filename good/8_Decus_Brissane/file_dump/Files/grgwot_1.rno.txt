.hl 1 INTRODUCTION
Program WOT is a monitor program for the RSX-11M computer system.
It allows a user with a VT100 terminal to observe the system
status on a dynamic basis, and in particular allows monitoring of
activity on the system checkpoint files.
.hl 1 USING WOT
The utility may be invoked by any of the standard RSX methods.
The task file resides in the [2,54] library, and is usually
installed with task name '...WOT'.

Any of the following commands may be used:
.s
.lm +5
.nf
WOT

RUN ...WOT

RUN LB:[2,54]WOT
.S
.LM 0
.f
If the program is used from a terminal which is not a VT100 a
string of apparently meaningless characters will be displayed.
.hl 1 INFORMATION DISPLAYED
The information displayed by the program includes the following:
.ls
.le Date and time.
.le Logged-on terminals.
.br
For each terminal currently logged on to the system the display
includes:
.ls
.le The terminal number;
.le The user's login UIC;
.le The user's current UIC;
.le The name of any task which has attached the terminal.
.els
For privileged users the UIC displayed in the second field is
the same as the current UIC, since on RSX-11M a SET#/UIC command
for such a user changes the parameters stored in the relevant
tables.
.le Pool statistics.
.br
The current state of the pool, or Dynamic Storage Region, are
displayed. Parameters shown include the total pool space, the
number of free blocks and the sizes of the smallest and largest
free blocks.
.le Active tasks.
.br
Active tasks are listed in two groups, showing which are
currently in memory, and which have been swapped out onto the
system checkpoint file or files.
Tasks are listed in order of decreasing priority.
.els
Because of the limits imposed by the screen size the lists of
active terminals and active tasks may be truncated.
In this case a status line will be displayed in the appropriate
area to indicate the number of other terminals or tasks which are
active.
.hl 1 CONTROLLING THE PROGRAM
Once started, the program runs continuously, updating the screen
about every two seconds.
Before each screen update a check is made of the input buffer of
the initiating terminal.

If a CTRL/C or a CTRL/Z is found the program terminates.
Any other key causes a complete screen re-write;
this can be useful if messages from another task or user have
destroyed the screen format.
