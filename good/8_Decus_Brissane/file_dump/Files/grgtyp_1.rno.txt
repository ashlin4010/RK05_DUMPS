.hl 1 INTRODUCTION
The TYP utility may be used to list files on an RSX-11M computer system
in a variety of formats.
It combines various features of the PIP and DMP utilities.
.hl 1 USING TYPE
The utility may be invoked by any of the standard RSX
methods. The task file resides in the [2,54] library, and is
usually installed with task name '...TYP'.

Any of the following commands may be used:
.s
.lm +5
.nf
TYPE

TYPE command-string

RUN ...TYP

RUN LB:[2,54]TYP
.lm 0
.s
.f
.hl 1 COMMAND FORMAT
The command string may be null, in which case TYPE prompts
the user for a command string.

The usual command format is:
.s
.lm +5
TYP filespec/switches,filespec/switches,...
.s
.lm 0
The meanings of the various parameters are described in
the following sections.
.hl 2 The file specification
.br
'filespec' is any legal file name.
If the user does not specify a file type, a default type of
'.MAC' is used.
.hl 2 Switches
.br
Switches may be used to request the utility to list files in
various alternative modes.

If no switches are specified the file is listed as a normal
text file.
This is the normal use of the utility.

Switches allow the user to inspect the file as octal words,
RAD50 encoded words, or as a string of ASCII characters.
Several combinations of switches are possible to allow
combined outputs.

The switches are:
.ls
.le /L - List mode
.br
This is the same as no switch.
.le /A - ASCII mode
.br
Dump the file as a string of ASCII characters.
Non-printing characters are replaced by dots (.).
.le /O - Octal mode
.br
Dumps the file as octal words and as ASCII characters.
This switch may be combined with either or both of two
further switches:
.ls
.le /A##- include ASCII characters
.br
This is the default for an octal dump.
.le /-A#- supress ASCII characters.
.le /R##- include RAD50 codes
.els
.els
Switches must be specified separately for each file.

Only the first character of any switch is examined.

Legal switch combinations are:
.s
.ts 3,13,23,33,43,53
.nf
	/L
	/A
	/O	/O/A	/O/-A	/O/R	/O/R/A	/O/R/-A
.f
.hl 1 RESTRICTION
The program assumes that files to be listed in list mode (/L
or no switch) are normal variable length records with an FD.CR
characteristic.
Certain system programs (for example, TKB and RNO) create files
which do not have ths characteristic, and the result of TYPing
such files is unpredictable.
