20-MAY-81
----------

	PROBLEM		THE PROGRAME CRASHED WITH AN ERROR MESSAGE FROM
			"FILSPT" WHEN ATTEMPTING TO LOCATE A RECORD WHEN
			THERE WAS MORE THAN 1 POSSIBLE MATCH.

	SOLUTION	CODING ERROR IN 'KEYMNG' WHICH IS NOW FIXED. RE-LINK
			ALL PROGRAMS.


	PROBLEM		THE PROGRAME CRASHED AFTER HITTING A <CR> AFTER
			EXITTING 'FORM ENTRY MODE'.

	SOLUTION	USED MTPS/MFPS INSTRUCTIONS WHICH ARE LSI-11 SPECIFIC
			AND DO NOT WORK ON PDP-11. CHANGE 'FEDMNG' CODE SO
			THAT THESE INSTRUCTIONS ARE NOT USED.


	PROBLEM		WHEN USING 'REPORT' WITH THE CONSTRUCT '$N$E' THE
			LAST RECORD IS NOT PRINTED OUT.

	SOLUTION	THIS IS NOT A "BUG". THE '$N' COMMAND CAUSES THE
			LAST RECORD TO BE READ IN, THEREBY DECREMENTING 
			THE 'COUNTER' TO ZERO, SO THAT WHEN THE '$E' COMMAND
			IS EXECUTED PHASE 2 IS ABORTED AND THE LAST RECORD
			IS NOT PROCESSED. THE '$E$N' CONSTRUCT SHOW BE USED
			INSTEAD.
